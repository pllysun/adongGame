# 03 · 宗门经营

玩家的主操作面。设计原则：**每一种资源都有明确的产出口和消耗口，每一座设施都明确回答
"它改哪个数字"**。

## 1. 资源体系

| 资源 | 产出 | 消耗 | 设计角色 |
|------|------|------|---------|
| 灵石 | 灵田余粮折算、矿脉、坊市贸易、秘境收获 | 月俸、建设、拍卖、阵法维护 | 万能货币，主预算 |
| 灵米 | 灵田 | 弟子口粮（按人头） | 人口硬约束 |
| 灵草 | 灵田（药圃区）、采集任务、秘境 | 炼丹原料 | 丹药供应链上游 |
| 材料 | 矿脉、妖兽尸体、秘境 | 建设、炼器 | 建设供应链上游 |
| 声望 | 弟子扬名、宗门战胜利、侠义事件 | 招收质量、坊市议价、外交事件选项；**每点声望月入 0.06 灵石（供奉收入，飞轮的经济端）** | 招收飞轮的轴 |
| 宗门气运 | 隐性累积（善行/福地/镇宗之宝） | 大机缘事件权重 | 隐藏彩蛋层 |

月度结算：`灵石净流 = 产出 - Σ(弟子月俸 × 待遇档) - 设施维护`。
**赤字不立即死人**，但欠俸触发心境惩罚与（贪婪/野心弟子的）叛逃检定——经济压力通过人事系统表达。

## 2. 设施体系

设施有等级（对应七阶颜色），升级消耗灵石+材料，部分需要前置境界的弟子坐镇。

| 设施 | 改哪个数字 | 备注 |
|------|-----------|------|
| 聚灵阵 | 全宗修炼速度倍率（核心升级轴） | 等级上限受灵脉品质约束（选址的长期回报） |
| 洞府群 | 弟子容量硬上限；高档洞府给个人修炼加成 | 灵穴洞府稀缺，分配权是玩家手里的胡萝卜 |
| 藏经阁 | 可收藏功法品阶上限；弟子借阅学功法 | 借阅权限是宗门规则的一部分 |
| 灵田 | 灵米/灵草产量 | 需弟子执勤，劳动力调度的主要去向 |
| 炼丹房 | 解锁丹方品阶；丹药产能 | 需炼丹师（悟性高的弟子转职培养） |
| 演武场 | 战斗经验增速；切磋事件场所 | 关系网事件高发地 |
| 护山大阵 | 防御战力倍率；危机事件的保险 | 平时纯支出，妖兽潮/敌袭时决定生死 |
| 静室 | 心境恢复速率 | 心魔治理的硬件 |
| 戒律堂（后期） | 叛逃/内斗检定难度 | 配合"记仇/野心"特质，秩序管理 |
| 炼器坊（后期） | 法宝锻造 | 法宝供应链 |

## 3. 招收体系

### 开山大会（主渠道）

- 周期：默认每 3 年一届（玩家可在 1~10 年间调整；越频繁单届质量越低）。
- 候选人数：`12 + 声望档位加成`，资质分布见 01 文档 §2。
- 流程：测灵（显示资质/灵根/悟性，隐藏心性/气运）→ 玩家勾选录取 → 入外门。
- 录取上限 = 洞府容量余量；**多收摊大饼 vs 精养天才**两种流派都要可行。

### 事件渠道（补充，气运与故事的入口）

| 来源 | 特点 |
|------|------|
| 散修来投 | 自带境界与功法，但性格隐藏信息多（可能带仇家事件链） |
| 路边孤儿 | 资质随机但气运均值偏高，养成系叙事 |
| 敌宗叛逃者 | 即战力，忠诚存疑（埋"卧底"事件链可能性） |
| 转世重修者（稀有） | 仙品资质的主要来源，自带宏大事件链 |

## 4. 职位体系

| 职位 | 人数 | 效果 | 任免 |
|------|------|------|------|
| 掌门 | 1 | 全宗士气；其资质/声望影响外交事件选项 | 玩家任命；老掌门坐化未指定继任 → 夺位事件链 |
| 长老 | 数名（按宗门规模） | 各掌一摊：执法（秩序）、传功（教学加成）、外务（贸易/外交） | 玩家任命，野心特质者落选会记恨 |
| 执事 | 内门弟子兼任 | 设施岗位效率 + | 按宗门规则自动选拔 |
| 真传 | 限额 | 资源待遇最高档，专属洞府 | 规则自动 + 玩家钦点 |

职位不只是 buff 槽——**任免本身产生人事事件**（争位、不服、提携亲信），是关系网的搅拌器。

## 5. 宗门规则（间接管理的控制面板）

玩家通过规则页签全局调控，规则改变弟子自主行为权重与系统门槛：

- **晋升规则**：内门门槛（默认炼气后期）、真传门槛（默认筑基+地品资质）——可改
- **待遇规则**：各级月俸档位、丹药配给优先级（按资质/按境界/按贡献）
- **功法借阅**：各级可借阅的功法品阶上限
- **行为约束**：是否允许私自游历／结道侣是否需要批准／执勤豁免名单
- **突破管理**：突破必须申请（玩家逐个审批）或自动放行（按成功率阈值）

> 默认值让"完全不管"也能运转（自动驾驶），让"事事插手"也有抓手——
> 玩家的参与度自己选。

## 6. 与外部世界的接口（事件化世界的挂载点）

第一版外部世界 = 一组**世界状态变量**（正魔局势、妖兽活跃度、临近三宗的敌意值）
+ 围绕它们的事件库。架构上这些变量与 NPC 宗门档案（名称/宗主/大致实力）已经建模，
后续版本把"档案"升级成"也在跑模拟的实体"即可，存档结构不变。
