# 07 · 技术架构

栈：Vue 3 + TypeScript + Vite + Pinia + Tailwind；后期 Tauri 打包桌面版。
**第一纪律：引擎是纯 TS，零 Vue 依赖，引擎状态永不进入 Vue 深度响应式系统。**

## 1. 目录结构

```

│   ├── engine/              # ⛔ 禁止 import vue/pinia —— ESLint 规则强制
```

## 2. SimEngine 门面

```ts
class SimEngine {
  constructor(seed: number, scenario: ScenarioConfig)
  tick(): void                          // 推进 1 游戏日
  command(cmd: PlayerCommand): Result   // 玩家指令唯一入口（录取/任命/建设/赐丹/规则变更…）
  getSnapshot(view: ViewQuery): Snapshot // UI 按需取浅拷贝切片
  on(channel: 'event'|'chronicle'|'combat'|'pause-request', cb): void
  save(): SaveBlob
  static load(blob: SaveBlob): SimEngine
}
```

- **指令模式**：UI 永不直接改世界状态，一切修改走 `command()`——
  这让存档回放、操作日志、未来的活世界 AI（NPC 宗门也发 command）全部免费获得。
- **快照模式**：UI 的 Pinia store 持有 `shallowRef(snapshot)`，引擎每 tick 批量结束后
  发"脏视图"通知，store 拉取新切片整体替换。Vue 只 diff 顶层引用，深层零追踪开销。

## 3. 时间与 tick 模型

- 最小时间单位：**1 游戏日**；系统按各自节奏挂在不同周期上（日/旬/月/年，见 §1 注释）。
- 速度档：1x = 3 日/秒，3x = 9 日/秒，10x = 30 日/秒（即 10x 下一游戏年 ≈ 12 秒）。
- 实现：`requestAnimationFrame` 驱动，按真实流逝时间折算应执行的 tick 数，
  单帧补帧上限 60 tick（防卡死后疯狂追帧）。
- 自动暂停：引擎发 `pause-request`（重大事件/突破时刻/战斗），UI 层执行暂停。

## 4. 确定性与随机

- 全引擎共用**种子化 RNG**（sfc32），禁止 `Math.random`（ESLint 强制）。
- RNG 按子系统分流（`rng.fork('combat')`），保证"同存档同操作必然同结果"，
  bug 复现和数值回归测试都依赖这一点。

## 5. 存档

- 格式：`{ version, seed, day, world: WorldState, commandLog? }`，JSON 可序列化。
- 位置：浏览器期 IndexedDB（多档位+自动存档轮换 3 个）；Tauri 期换文件系统，格式不变。
- **版本迁移**：每次 WorldState 结构变更写一个 `migrations/vN.ts`，旧档逐级升级。
  从 M1 起就执行，不留"以后再说"的债。

## 6. 无头数值验证（这个项目的秘密武器）

```bash
npm run sim -- --years 800 --runs 50 --report
```

`src/headless/` 下的 Node 运行器：跳过 UI 直接驱动引擎 + 一个"自动玩家"策略脚本
（按默认规则招人/建设/应战），批量跑 N 局 × M 年，输出统计报告：

- 各资质弟子的境界分布曲线（凡品有没有奇迹？天品是不是必成？）
- 首个金丹/元婴/渡劫的出现年代分布
- 宗门灭亡率与灭亡原因占比
- 经济曲线（灵石何时开始失衡）

**每次改数值常量必须跑回归**。02 文档里所有数字都只是初值，真正的平衡靠这里迭代。
M4 里程碑起 CI 化（每次提交跑 10 局快速版）。

## 7. 性能预算与逃生门

- 目标规模：200 弟子 / 10x 速度（30 tick/秒）流畅；预算内做法：行为/关系/事件都在
  旬/月级别结算，日级只有修为加法——单 tick 实际计算量极小。
- 若未来卡顿：引擎整体搬进 **Web Worker**（门面已是消息式 API，搬迁成本低）。
- 若未来要地图视图：主视图区嵌 Canvas/PixiJS 组件，不动整体架构。

## 8. 工程规约

- ESLint + Prettier + strict TS；`engine/` 内禁 import UI 层（依赖方向单向）。
- 单文件软上限 500 行（已有项目纪律），系统模块超限即拆。
- 引擎核心系统（突破、战斗、事件选角）必须有 vitest 单测，内容数据用 schema 校验
  （zod）防手误。
