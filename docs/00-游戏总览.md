# 00 · 游戏总览

> 暂定名：《问鼎仙途》（占位，待定）
> 类型：上帝视角 · 修真宗门模拟 · 文字+立绘
> 参考系：《了不起的修仙模拟器》《RimWorld》《太吾绘卷》（人事涌现部分）

## 一句话定位

玩家扮演冥冥中的"天道之手"，创立一座宗门，招收凡人弟子，看着他们修炼、争斗、相爱、陨落，
一代代传承数百年，最终培养出多名渡劫大能，举宗飞升仙界。

## 设计支柱（所有决策的最高裁判）

任何系统设计有分歧时，回到这三条判断：

1. **故事自己长出来**——系统之间咬合产生的涌现事件（弟子殉情、天才陨落、叛徒复仇）比写死的剧本更珍贵。
2. **玩家是天道不是保姆**——定规则、分资源、关键时刻落子；不微操弟子的每一步。
3. **数百年的长跑感**——寿元、世代更替、宗门兴衰是节拍器；单个弟子会死，宗门的故事不停。

## 已确定的核心决策

| 决策点 | 选择 |
|--------|------|
| 技术栈 | Vue 3 + TypeScript + Vite + Pinia + Tailwind；后期 Tauri 打包 |
| 架构 | 纯 TS 无头模拟引擎与 UI 彻底分离；引擎状态不进 Vue 响应式系统 |
| 控制方式 | 间接管理为主（定规则/分资源/派任务），关键时刻可介入 |
| 外部世界 | 第一版事件化（秘境/妖兽潮/敌宗皆为事件），架构预留活世界接口 |
| 弟子深度 | 性格特质 + 关系网，行为受性格驱动 |
| 时间推进 | 连续推进 + 暂停 + 变速（1x/3x/10x），重要事件自动暂停 |
| 战斗 | 全自动结算，输出叙事化战报；玩家博弈在战前配置 |
| 稀有度 | 全游戏统一七阶（颜色为视觉层，各域有自己的等级名称） |
| 立绘 | 纸娃娃分层组合，气质随境界变化 |

## 三层核心循环

```
弟子层（日/月结算）：修炼 → 修为增长 → 瓶颈 → 突破判定 → 晋级/受挫/陨落
                     ↑ 资质·功法·丹药·心境·设施共同决定效率与成败
宗门层（玩家主操作）：招收 → 分配资源 → 建设施 → 定规则 → 应对事件
世界层（压力与机遇）：秘境·妖兽潮·敌宗·天材地宝·正魔大势 → 以事件形式涌入
```

## 游戏开局

1. **选址**：3~5 个候选灵脉，品质（修炼倍率）与危险度（妖兽/敌对势力事件权重）成正比，
   一开始就是风险偏好抉择。
2. **祖师**：开局赠送一位金丹初期老祖（随机资质/性格/功法），是前期唯一战力支柱，
   也是第一代掌门。老祖的寿元（约 500 年）就是前期的隐形倒计时。
3. **建宗**：命名宗门、获得初始灵石与三名随机资质的种子弟子，时间开始流动。

## 胜利与失败

- **胜利**：宗门同时拥有 3 名渡劫期大能，且完成"飞升大典"事件链 → 举宗飞升结局
  （按编年史生成本局宗门史诗回顾）。
- **失败**：弟子归零，或护山大阵被攻破且无人幸存 → 灭门结局（同样生成编年史，败局也是故事）。
- **软失败螺旋**：声望扫地 → 招不到人 → 青黄不接 → 高境界长老老死 → 加速衰亡。
  允许玩家从谷底翻盘，这是模拟器最好看的剧情。
