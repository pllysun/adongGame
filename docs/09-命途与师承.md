# 09 · 命途与师承系统

> 本篇是迄今最大的单一系统设计，目标是把游戏从"事件抽奖机"升级为"一群有持久人生的个体"。
> 状态：**设计已锁定，待分阶段实现**（M8+）。实现前以本文为准。

## 0. 要解决的问题

当前事件是**无状态的一次性函数**：effect 改完 `mood`/修为就结束，下一次这个人是一张白纸。
表现就是"换个人名再抽一次、下次就没事了"。本系统要把它变成**有状态、会累积、可回溯**的个体人生：

1. **持久累积**——每次抉择给角色留下不可逆的痕迹，而非一次性数值波动
2. **可回溯因果**——能查看"这个人为什么变成今天这样"，每个决策的后果都有据可查（经历树）
3. **个体能动性**——每个角色有自己的 AI，面对同一件事因各自经历做出不同选择，长出不同人生

## 1. 已锁定的设计决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 个人抉择由谁做 | **角色 AI 自主，玩家间接影响** | 贴合"每个人有自己 AI"与"天道不是保姆" |
| 内心可见度 | **模糊档位 + 征兆**，不给精确数值 | 延续信息迷雾，"识人"是玩法不是仪表盘 |
| 心相轴粒度 | **五轴**：道心/心魔/魔倾/忠诚/声名 | 够区分人格又不失控 |
| 经历树范围 | **数据全记，UI 突出转折点** | 几百年下来翻得动 |
| 决策内核 | **效用 AI（Utility AI）** | 一个连接器，避免 N×M 硬编耦合 |
| 学习方式 | **不用神经网络/ML**，用无头模拟调权重 | 确定性可复现、可解释、轻量、可作者化 |
| 峰系统野心 | **半涌现**：7 类固定职能峰，可生灭分争，道统树留扩展位 | 先落地后扩展 |
| 玩家对峰/师徒权限 | **天道哲学**：批准开峰/分配资源/调解峰争/钦点峰主 | 一贯哲学 |

## 2. 算法底座：一个"连接器"而非 N×M 根硬编线

**核心原则：不去一对一硬编系统耦合，而是让每个系统往同一个效用函数里"投考量"，
决策函数自动成为所有系统的交汇点。**

```
心动值(选项) = Σ 考量ᵢ × 权重ᵢ(选项)
  · 考量可来自任意系统：心相轴、关系网、宗门资源、境界、所在峰、师承、世界局势……
  · 按 softmax(心动值 / 温度) 抽取；温度 = f(心性)：心性高→冷静选优，心性低→冲动随机
要接入新系统 = 注册一条考量，不改决策代码。
```

| 算法 | 用在哪 | 备注 |
|------|--------|------|
| 效用 AI + 响应曲线 | 角色一切自主抉择；响应曲线把原始值（如心魔值）映射成 0-1 考量 | 可作者化、可解释、可调（《Behavioral Mathematics for Game AI》范式） |
| softmax + 温度=心性 | 把"性格决定决策风格"变成一个参数 | 一行代码，质变手感 |
| 标签产生式规则 | 记忆打标签 → 未来事件按标签门控加权（涌现因果） | 迷你前向推理机 |
| 并查集 / 标签传播 | 峰的形成与师承聚类（见 §10） | 确定性、轻量 |
| 树上数值传播 | 师尊心相/特质沿师承树向下漂移徒弟（门风涌现） | 加权 DFS |
| 因果 DAG | 经历树（记忆节点 + causedBy 边），"为什么黑化"= 祖先回溯 | 图遍历 |
| 戏剧导演（张力预算） | 监控宗门叙事张力，动态调事件/征兆权重——别让悲剧扎堆、冷场撒火星 | 让模拟自调节奏（参 L4D AI Director / 交互叙事 drama manager） |

**为何不用 ML**：要确定性可复现（同存档同结果，否则经历树与回归测试全崩）、要可解释
（经历树须能讲清因果，神经网络是黑箱）、要轻（整包 gzip ~89KB）、要可作者化（审数据即改行为）。
"训练循环"就是已有的**无头模拟**——蒙特卡洛跑数百局来标定效用权重。

> 理论彩蛋：玩家从"模糊档位+征兆"去猜内心，本质是**部分可观测系统的状态估计**（POMDP 信念维护）。
> 这给信息迷雾一个正式依据——游戏卖的就是"识人"这件估计活。

## 3. 命途系统四层

```
第一层 阅历记忆   memories[]    经历树的数据 + 未来逻辑的可查询底座
第二层 五轴心相   beliefs       事件永久写入的"自我"，慢漂移、不回弹
第三层 动态特质   traits 升级    经历可习得/失去特质 = 经历树的"质变节点"
第四层 角色 AI    utility       自主决策，玩家只能间接影响
```

### 3.1 阅历记忆

每个弟子挂 `memories: Memory[]`。一条记忆结构化、可查询、可连因果：

```ts
interface Memory {
  day: number
  eventId: string
  kind: 'turning' | 'minor'      // 转折点 vs 琐事（UI 用）
  text: string                    // 已插值的"他的事"
  choice?: string                 // 当时的抉择（角色自己选的）
  impact: Partial<Beliefs> & { traitGained?: string; traitLost?: string }
  tags: string[]                  // 如 丧侣 弑师 受恩 背叛 顿悟 —— 未来逻辑据此门控
  causedBy?: number               // 指向更早一条记忆的 day/id，形成因果链
}
```

### 3.2 五轴心相（与会回弹的 `mood` 区分：心相是长期自我）

| 轴 | 含义 | 谁写它 | 谁读它 | 漂移 |
|----|------|--------|--------|------|
| 道心 | 信念/定力 | 悟道、坚守、渡劫 | 突破率、抗心魔、角色 AI | 缓慢，受心性 |
| 心魔/执念 | 创伤累积 | 丧侣、被掳、弑杀、屡败 | 心魔事件门槛、突破惩罚；可被"了结心结"消解 | 只升难降 |
| 魔倾 | 个人正魔倾向 | 修魔功、行杀伐 vs 行善 | 角色 AI、黑化路径、魔教/正道事件 | 双向 |
| 忠诚 | 对宗门 | 师恩、提携 vs 欠俸、被弃 | 叛逃判定、出力程度 | 双向 |
| 声名 | 个人名望 | 扬名/丑闻 | 外部事件找上门的概率 | 双向 |

初值由资质/灵根/出身特质派生；漂移速率受心性与设施（静室/戒律堂）调制。

### 3.3 动态特质

特质不再只是出生抽到的。被同门出卖多次→习得「多疑」；屡屡补刀→「嗜杀」；剑冢顿悟→「剑心通明」。
**特质的获得/失去就是经历树上的"质变节点"**，是人物弧光最直观的体现。
（既有 30+ 静态特质保留；新增一批"可后天习得"标记的特质。）

### 3.4 角色 AI

见 §2 效用公式。要点：
- 个人抉择事件**不弹给玩家**，由角色 AI 自行抽取选项
- 每个选项的"心动值"由心相轴 + 特质 + 相关记忆标签 + **玩家影响修正项**加权而成
- `玩家影响修正` 是所有间接手段的数学入口（见 §4）
- 抉择后果永久写心相轴 + 记一条 Memory（可能习得/失去特质）

## 4. 玩家主场：影响力工具箱 + 干预窗口

角色自主后，玩家的乐趣从"替角色选"变成**"读懂人、在对的时机往对的人身上下注"**。

| 层级 | 手段 | 怎么生效 |
|------|------|---------|
| 环境（被动全局） | 门规、待遇、静室、戒律堂、聚灵阵 | 改心相轴漂移速率与抉择权重 |
| 物品（主动点对点） | 赐静心丹压心魔、赐法宝功法涨忠诚、安排道侣师徒 | 直接写心相轴 / 建关系 |
| 人事（主动） | 派长老护道、调岗、收徒、逐出 | 触发"开解心结"类事件、改成长环境 |
| **机缘分配**（稀缺杠杆） | 秘境名额、灵乳、造化玉髓给谁 | 决定谁能兑现潜力——塑造命途最强的手 |
| **天道亲临**（保底大招） | 极稀缺地直接扭转一次重大抉择 | 消耗**气运** + 冷却，强行覆盖角色 AI；代价高、有限 |

**征兆-干预窗口**：角色做出激烈抉择前先放**征兆事件**——

> "李慕云近来形单影只，眼底渐有戾气，长老们看在眼里。"

给玩家数月窗口去赐丹/护道/调岗，赶在心魔劫自动触发前改其心相轴。
**征兆清晰度受角色性格影响**：孤僻/阴鸷者征兆隐晦，更容易看走眼——既不坑人又保留意外。

玩家核心循环：**读征兆 → 判断谁值得救/救得动 → 把有限影响力下注在对的人身上**。

## 5. 信息迷雾

心相轴只给**模糊档位**（"道心坚定/动摇""似有心结""暗藏戾气"）+ 征兆线索，不给精确数字。
与既有"资质灵根可见、心性气运隐藏"一脉相承。立绘神态随心相档位变化，是另一条暗示通道。

## 6. 经历树（弟子详情新增「生平」页）

数据是因果 DAG，UI 是带因果链的竖向时间轴；转折点高亮，琐事折叠。

```
张三 · 命途
├─ 天玄历48年 拜入山门（外门）
├─ 天玄历71年 ❤ 与林霜结为道侣
├─ 天玄历89年 ⚔ 林霜战殁于妖兽潮            ← 转折点
│    └─因→ 道心-30 心魔+40 习得「孤僻」
├─ 天玄历92年 🌀 心魔劫·堕道（因上一条）
│    └─因→ 修《血影魔功》 魔倾+25 习得「嗜杀」
├─ 天玄历95年 🗡 手刃仇人王五
└─ 天玄历97年 🌫 叛出宗门                    ← 命途终
     └─溯源链：林霜之死 → 堕道 → 嗜杀 → 叛逃
```

离场角色（叛逃/战死）保留经历树可查——宗门反派能一路回溯来历，宗门史诗长在个体命途上。

## 7. 师徒系统

- 拜师 = 一条 `master` 关系边 + 一个峰的归属（内门弟子必有师尊；外门无）
- **师尊的心相与特质沿师承树向下漂移徒弟心相**（§2 树上传播）：杀伐师尊带出戾气门徒，
  正直师尊抬高门下道心。几代传下，**一个峰长出自己的"门风"**——涌现且可回溯
- 师徒是命途高能节点：师尊亡故→徒弟心结；师尊被害→**弑师/复仇**；青出于蓝→师尊与有荣焉
- 嫡传（真传）/ 记名弟子 = 峰内更细的师承层级

## 8. 峰系统：宗门的"组织演化"

### 8.1 四阶段生命周期（随宗门实力自动推进）

| 阶段 | 触发 | 组织形态 |
|------|------|---------|
| 草创 | 开局 | 无内外门，人人"同门"，老祖非正式地教所有人 |
| 立宗 | 人口 ≥ N 且 出金丹 | 正式分内/外门，动态入内门门槛出现 |
| 分峰 | 多名化神+ 且 弟子够多 | 强者**开峰立派**，弟子按师承/灵根/交情聚入各峰 |
| 道统演化 | 持续 | 峰新建/分裂/衰亡/互争（道统树为半涌现扩展位） |

### 8.2 动态入内门门槛 = 反馈控制器

```
门槛 = controller(宗门发展度)，发展度 = f(顶级境界, 人口, 峰数)
早期：门槛 = 0（人人同门）
壮大：资质低者须熬到筑基方入内门；资质高者被各峰抢收、直入内门
```

天然演出"天才一进山门被疯抢，庸才得熬资历"。

### 8.3 七类职能峰（半涌现：固定类型，可生灭）

剑峰（战力主力）/ 丹峰（炼丹房）/ 器峰（炼器坊）/ 阵峰（护山大阵）/
御兽峰（妖盟）/ 传功峰（教学加成）/ 外门管事峰（经济）。
**职能绑定设施**，机制上各不相同；峰有峰主、镇峰之宝、一脉道统（招牌功法+理念）。

### 8.4 涌现机制（并查集 / 标签传播）

- **开峰条件**（其一）：高境界+足够威望与门徒 / **自创新功法**（开新道统）/ 某职能登峰造极
- **聚类**：开峰时跑一次社区划分——以高威望者为种子，弟子按"最强师承/灵根契合/交情"边并入某峰
- **会动**：峰主无嗣而亡→衰亡；理念分歧→**分峰**；真传名额/资源→**峰争**；
  弟子叛离→**跳峰**（带背叛味）；强者出走→**分宗**（叛徒在外立成敌宗 = 反派制度化）
- 峰主继承/峰争用沿师承树的**威望聚合**裁决

### 8.5 玩家角色（天道哲学）

峰自形成自运转，玩家只：批准开峰、给各峰分配资源、调解峰争、钦点峰主。不直接任命/指派/强分。

### 8.6 修真扩展位（暂留接口）

道统树（功法谱系像物种演化树分叉竞争断绝）、欺师灭祖/清理门户、传承之宝失传与被夺。

## 9. 与现有系统的咬合（串联而非另起炉灶）

- `mood` 留作短期近况；心相轴是长期自我
- 既有 `traits` 升级成可演化；既有 `relationships` 是人际记忆，心相轴是自我记忆
- 既有 `chronicle`（宗门编年史）与个人经历树互 link——宗门大事从某人命途冒出
- `behavior.ts`（自主行为）、突破、战斗都从心相轴读数：心魔高→易走火、忠诚低→易游历不归
- 既有 `flags` 战略轴（demonLean/ally）与个人魔倾/声名分层：宗门级 vs 个人级

## 10. 数据结构草案（增量、向后兼容）

```ts
interface Beliefs { daoxin: number; xinmo: number; molean: number; loyalty: number; fame: number }

interface Disciple {
  // …既有字段…
  beliefs: Beliefs
  memories: Memory[]
  masterId: string | null         // 师尊（外门为 null）
  peakId: string | null           // 所属峰（未分峰为 null）
  lineageTier?: 'zhuan' | 'jiming' // 嫡传/记名
}

interface Peak {
  id: string
  name: string
  type: 'sword' | 'alchemy' | 'forge' | 'array' | 'beast' | 'teach' | 'outer'
  founderId: string
  masterId: string                // 当代峰主
  signatureGongfa?: string        // 镇峰道统
  treasure?: string               // 镇峰之宝
  prestige: number                // 威望聚合，用于峰争/继承
}

interface WorldState {
  // …既有…
  peaks: Peak[]
  sectStage: 'founding' | 'sect' | 'peaks'
  innerThreshold: { realm: number; aptitude: number }  // 动态门槛
}
```

## 13. 风险与开放项

| 风险 | 缓解 |
|------|------|
| 效用权重难调（角色全在黑化或全是圣人） | 无头模拟从 P3 起标定，看"黑化率/叛逃率/道心分布" |
